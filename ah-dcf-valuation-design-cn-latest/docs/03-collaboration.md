# 03 协作规范

## 1. 组织与角色

项目采用“产品 + 量化 + 数据 + 平台 + 模型风险”协同机制。

- 产品/投研：定义业务规则、阈值、使用场景、评审口径
- 量化研究：定义 DCF 方法、参数规则、情景框架
- 数据工程：保证数据接入、标准化、质量闸口
- 平台工程：保障服务稳定、接口可用、发布运维
- 模型风险管理：独立验证、挑战测试、合规审查

## 2. RACI 责任矩阵（简化）

- 需求定义：A=产品，R=产品/投研，C=量化，I=工程
- 模型方法：A=量化负责人，R=量化，C=模型风险，I=产品
- 数据治理：A=数据负责人，R=数据工程，C=量化，I=平台
- 系统发布：A=平台负责人，R=平台工程，C=数据/量化，I=业务
- 风险审查：A=模型风险，R=模型风险，C=量化/产品，I=平台

## 3. 研发流程规范

## 3.1 需求到发布流程

1. 需求评审：确认目标、口径、验收标准
2. 设计评审：架构、数据、模型、风险联审
3. 开发实现：按模块推进并保持文档同步
4. 测试验收：功能、数据、回测、压力、审计
5. 灰度发布：小范围放量，监控稳定后全量
6. 复盘优化：沉淀问题与改进项

## 3.2 变更管理

- 任何模型假设变更必须走变更单
- 变更单包含：原因、影响范围、回滚方案、验证结果
- 重大变更需双审批（业务 + 模型风险）

## 4. 文档与审计规范

- 文档最小集合：需求文档、架构文档、数据字典、参数字典、发布记录、事故复盘
- 估值结果必须关联：
  - 数据版本 ID
  - 参数版本 ID
  - 模型版本 ID
  - 运行批次 ID
- 审计留痕保留周期按机构内控要求执行

## 5. 质量规范

- 评审规范：关键变更至少 2 人评审（Review）
- 测试规范：单测/集成/回测/回归测试清单化
- 发布规范：持续集成通过（CI 全绿）+ 发布检查单 + 回滚演练
- 事故规范：分级响应 + 48 小时内复盘

## 6. 沟通机制

- 日常：每日站会（进度、阻塞、风险）
- 周会：版本计划、关键指标、遗留问题
- 月度：方法论回顾与业务价值复盘
- 应急：告警触发后建立临时响应群，指定事故负责人（Incident Owner）

## 7. 协作规范要点总结

- 角色清晰：责任明确、边界明确
- 流程闭环：需求 -> 发布 -> 复盘全链路
- 治理可执行：变更、审计、质量、应急均可落地
