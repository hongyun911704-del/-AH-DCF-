# 04 路线图、风险与验收

## 1. MVP 路线图

## 阶段一（0-6 周）：最小可用版本

目标：

- 覆盖核心样本池（如前 300 只 A/H 标的）
- 实现基准情景（Base）DCF 自动估值日更
- 建立基础质量监控和估值查询

验收：

- 日批成功率 >= 98%
- 覆盖率 >= 95%
- 关键字段缺失率达标

## 阶段二（7-12 周）：全市场扩展

目标：

- 覆盖全市场 A/H 标的
- 引入乐观/悲观情景（Bull/Bear）与终值双方法
- 完成估值偏离告警机制

验收：

- 全市场稳定日更
- 异常估值识别能力稳定
- 投研使用率持续增长

## 阶段三（13-20 周）：治理与优化

目标：

- 完善解释引擎与归因视图
- 强化模型风险管理与审计闭环
- 建立版本化运营机制

验收：

- 估值结果可追溯率 100%
- 重大异常可定位并快速回滚
- 形成标准化发布节奏

## 2. 主要风险与应对

## 2.1 数据风险

- 风险：源数据延迟、口径冲突、A/H 映射错误
- 应对：多源校验、主键映射治理、延迟告警与回退机制

## 2.2 模型风险

- 风险：参数漂移、行业不适配、极端市场失真
- 应对：参数边界、行业模板、压力测试、人工复核白名单

## 2.3 系统风险

- 风险：批处理失败、接口不稳定、缓存污染
- 应对：任务重试、多级降级、蓝绿发布、快速回滚

## 2.4 组织风险

- 风险：跨团队目标不一致、流程执行不到位
- 应对：RACI 固化、检查单机制、周度治理例会

## 3. 核心 KPI 体系

- 覆盖 KPI：可估值标的覆盖率
- 稳定 KPI：日批成功率、失败恢复时长
- 质量 KPI：关键字段空值率、异常值率
- 业务 KPI：投研使用率、告警有效命中率
- 治理 KPI：审计通过率、变更合规率

## 4. 最终验收标准

判定“完整设计方案达标”的最低标准：

1. 架构完整且具备生产可用性说明
2. 数据流程具备质量与异常处理闭环
3. 协作规范具备明确角色、流程与治理机制
4. 路线图可执行，阶段目标与指标可量化
5. 方案不包含具体代码实现，仅含设计层交付物
