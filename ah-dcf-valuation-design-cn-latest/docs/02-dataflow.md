# 02 数据流程设计

## 1. 全链路流程

全流程分为六个阶段：

1. 数据采集
2. 数据标准化
3. 因子与特征构建
4. 预测生成
5. 折现估值
6. 发布与分发

## 2. 各阶段输入、处理、输出

## 2.1 数据采集

输入：

- 行情：收盘价、市值、成交量、复权因子
- 财务：三大报表、附注指标
- 预期：一致预期、目标价、盈利预测
- 宏观：利率、通胀、汇率、风险溢价

处理：

- 全量 + 增量采集
- 源头时间戳、文件校验、重复去重

输出：

- 原始数据分区（按日期、市场、来源）

## 2.2 数据标准化

处理：

- 证券代码归一（A/H、主次上市、历史代码）
- 会计口径映射（IFRS/PRC GAAP 关键字段对齐）
- 币种与汇率折算（CNY/HKD）
- 财报周期对齐（年度、半年度、季度）

输出：

- 标准实体表（发行人/证券/交易日历/币种）
- 统一口径财务宽表

## 2.3 特征构建

处理：

- 历史增长率、利润率、ROIC、再投资率
- Beta 回归特征、资本结构特征
- 行业中位数与分位数特征

输出：

- 估值特征集（特征数据集市）

## 2.4 预测生成

处理：

- 行业模板 + 历史趋势 + 外部预期融合
- 三情景参数扰动（增长、利润、折现率、终值）

输出：

- 未来现金流预测集（基准/乐观/悲观）

## 2.5 折现估值

处理：

- WACC 计算与边界校验
- 终值双方法并行估计
- 股权价值、每股内在价值、估值区间生成

输出：

- 每标的估值快照（带参数与解释）

## 2.6 发布与分发

处理：

- 写入服务层数据库
- 刷新接口缓存（API Cache）
- 推送告警（估值偏离阈值）

输出：

- 可查询、可订阅的日度估值结果

## 3. 数据质量与闸口

每阶段必须通过质量闸口：

- 完整性：关键字段空值率阈值控制
- 一致性：同一实体跨源数据冲突校验
- 合理性：参数上下界、同比环比异常检测
- 时效性：数据延迟超过阈值自动告警
- 可回溯：输入与输出均记录版本号和时间戳

## 4. 监控与告警

- 管道监控：任务成功率、耗时、延迟
- 数据监控：缺失、异常波动、结构漂移
- 模型监控：估值跳变、参数漂移、场景失衡
- 业务监控：估值覆盖率、接口调用率、告警命中率

## 5. 降级策略

- 一级降级：单源失败时切换备份源
- 二级降级：当日未完成时回退“上次可用快照”
- 三级降级：禁用高风险情景，仅保留基础估值并打风险标签

## 6. 数据流程设计要点总结

- 链路完整：采集到发布全链路闭环
- 质量可控：闸口、监控、告警、降级齐备
- 结果可信：标准化、可追溯、可复盘
