# 01 架构设计

## 1. 目标与原则

### 1.1 业务目标

- 覆盖 A 股 + H 股全市场标的的 DCF 自动估值
- 每日稳定产出估值区间、隐含收益空间、关键假设解释
- 支撑选股、跟踪、预警与复盘

### 1.2 设计原则

- 分层解耦：数据、模型、服务、治理独立演进
- 可解释：每个估值结果可追溯至输入、参数、版本
- 可运营：具备 SLA、监控、告警、降级与回滚机制
- 可扩展：可从 DCF 扩展到多因子/相对估值融合

## 2. 总体架构

系统采用“六层架构”：

1. 数据源层：行情、财报、宏观、无风险利率、行业参数、分析师一致预期
2. 接入层：批处理采集 + 增量更新 + 校验
3. 数据层：原始层 / 标准层 / 特征层 / 服务层 四级数据域
4. 模型层：FCFF/FCFE、WACC、终值、多情景估值
5. 服务层：估值接口（API）、筛选查询、订阅告警、结果解释
6. 治理层：元数据、血缘、版本、审计、权限、合规

## 3. 核心模块设计

## 3.1 实体映射模块（A/H 一体化）

- 统一公司主键（发行人标识）打通 A/H 双地上市主体
- 统一币种、会计口径、财报周期、交易日历
- 处理重组、改名、退市、停牌等事件

输出：标准证券主数据与公司映射关系。

## 3.2 假设参数模块

- 无风险利率曲线：按市场（A/H）与期限管理
- 股权风险溢价：按市场与行业区分
- Beta：滚动回归 + 下限/上限截断
- 永续增长率 g：按宏观约束和行业边界配置
- 资本结构：目标负债率 + 市值/账面混合口径

输出：按日期生效的参数快照（可版本化）。

## 3.3 现金流预测模块

- 预测收入增长、利润率、税率、折旧、资本开支、营运资本
- 支持行业模板（金融、周期、消费、科技等）
- 支持三情景：基准情景 / 乐观情景 / 悲观情景（Base / Bull / Bear）

输出：未来 N 年自由现金流路径。

## 3.4 折现与估值聚合模块

- 计算 WACC 并执行折现
- 并行计算两类终值（永续增长法 / 退出倍数法）
- 输出估值区间、分位值、相对现价偏离

输出：每标的估值结果与估值解释结构化记录。

## 3.5 结果服务模块

- 查询接口：按股票、行业、市场、日期、情景检索
- 订阅接口：触发“估值偏离阈值”告警
- 回放接口：历史快照复盘与假设变更对比

输出：投研终端可消费的估值服务。

## 4. 非功能要求

- 可用性：日批核心链路成功率 >= 98%
- 时效性：T+1 08:00 前完成全市场刷新
- 性能：单标的查询 P95 < 300ms
- 可审计：结果可追溯到数据版本、参数版本、模型版本
- 安全性：权限分层（只读/研究/运维/管理员）

## 5. 架构设计要点总结

- 结构完整（层次、模块、边界清晰）
- 能力闭环（采集 -> 建模 -> 输出 -> 治理）
- 面向生产（SLA、监控、审计、权限）
